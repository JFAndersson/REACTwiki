    <I. Vad är react?/>

    React är ett javascriptbibliotek som används för att skapa interaktiva användargränssnitt och hantera uppdateringar av existerande innehåll på ett effektivt sätt. Det gör detta genom att använda sig av en virtuell DOM (Document Object Model), som är en representation av ett HTML dokument i minnet, och som gör det möjligt att rendera och uppdatera komponenter på skärmen på ett smidigt sätt. Komponenter i React är rena, isolerade enheter som innehåller statisk information och logik, och som kan renderas på olika ställen i en webbapplikation. React utvecklades ursprungligen av utvecklare på Facebook och Instagram för hantering av deras nyhetsflöden, men släpptes sedan offentligt på GitHub i maj 2013 och har sedan dess använts av tusentals webbutvecklare världen över. React får fortfarande regelbundna uppdateringar och har blivit en av de mest populära teknikerna för att skapa interaktiva användargränssnitt på webben. För att kunna använda React krävs det att man har grundläggande kunskaper i javascript, eftersom det är det språk som används för att skapa de element som användaren ser på sin skärm. För att möjliggöra en enkel integrering mellan javascript och HTML används en viss syntax som kallas för JSX, som är en javascript-extension som gör det möjligt att skriva html kod direkt i javascript dokument. JSX är inte nödvändigt att använda, men det förenklar arbetsprocessen avsevärt och följer automatiskt med i alla installationer av React.

    1. React komponent med JSX (mindre text)

                                    function Hello(props) {
                                 return <div>Hello, {props.toWhat}</div>;
                                }
                                
                                const root = ReactDOM.createRoot( document.getElementById('root'));
                                root.render(<Hello toWhat="World"/>);

                                    function Hello(props) {

                                }

                                root.render(React.createElement( Hello, {toWhat: 'World'}, null));

    <III. Reacts komponenter och DOM/>

    React använder komponenter, vilket är javascript funktioner som returnerar HTML kod, för att skapa anpassningsbara och återanvändbara gränssnitt. Komponenterna är små, isolerade och hanterar sin egen logik. Detta gör det möjligt för programmeraren att bygga stora och komplexa webbapplikationer med React. Att dela upp sin kod i komponenter gör det också enklare för utvecklaren att felsöka sitt projekt då problem uppstår, eftersom komponenters länkar och följaktligen innehåll enkelt kan uteslutas från det objekt som man vill ska renderas till DOM:en. Din DOM, som tidigare beskrivet, står för "Document Object Model" och representerar hierarkin av all HTML kod som visas i din applikation. I React används rot-div:en som en container för att innesluta React-komponenter. Denna div är ansluten till webbsidans DOM och det är där React-komponenterna renderas och visas för användaren. Rot-div:en är därför viktig för att förena komponenterna med webbsidan och göra det möjligt för dem att renderas. Denna rot kan du se nedanför markerad genom den gula fyrkanten.

    <IV. Hur man skapar och renderar komponenter/>

    En viktig aspekt av React utveckling utgår på att man hierarkiskt strukturerar sin kod och följaktligen komponenter in i olika child-dokument. Dessa dokument vill man sedan ansluta till index.js, applikationens parent-fil som befinner sig högst upp i hierarkin, och därigenom rendera dem till DOM:en. Att arbeta med React medför således en hierarkisk process som man vill vara uppmärksam kring. Säg exempelvis att jag vill rendera en bild och en text under bilden i mitten av skärmen. Vad jag först behöver göra i så fall är att skapa ett nytt javascript dokument som jag väljer att döpa till "cardView". Detta gör jag genom att först klicka på mappen "src", och sedan klicka på filikonen med ett plustecken i VS Codes vänstra spalt för att skapa en fil där. Sedan skriver jag cardView.js, varvid "js" taggen på slutet av namnet gör filen till en javascriptfil.

    1. Figur XVII, 2. Figur XVIII

    Inuti cardView.js börjar jag sedan med att skriva en normal javascript funktion. Denna funktionen kan vi kalla "CardWithText". Notera hur jag har valt att skriva funktionens första bokstav som en versal. Detta är vad som på utsidan definierar skillnaden mellan vanliga javascript funktioner och komponenter. Den inre skillnaden är att komponenter alltid returnerar HTML, medan funktioner hanterar mer specifik logik. Dessutom har jag fört in props (properties) som parameter i komponenten, vilket representerar ett objekt som vi sedan hämtar relevanta värden ifrån. Dessa värden förs naturligt in komponenten när den kallas i parent-dokumentet. Jag börjar i alla fall med att skriva ut den tomma komponenten:

    <V. Hur förbereder man en databas-server för React?/>

    Serverkommunikation är en mycket viktig del inom React utveckling. Först och främst ger den dig möjlighet att bygga applikationer som kan visa dynamisk data - till exempel om du bygger en social media-applikation och du hämtar de senaste inläggen från en server. Utan denna kommunikation skulle din applikation endast kunna visa statisk information som är inbyggt i programmet. Det vill säga att applikationer och hemsidor som är dynamiska, data-drivna och kan interagera med andra system och API:er (99% av internet) inte hade kunnat utvecklas. I den följande genomgången kommer jag därför att förklara hur du kan bygga ett sådant system.

    Första steget blir således att skapa serverns projektmapp som du sedan vill placera i samma rot-mapp som din react applikation. Detta kan du göra smidigt i VS Code genom att klicka på mappikonen med en plussymbol. Denna ikon dyker upp längst upp i vänsterspalten (filträdsutforskaren) när din datormus hovrar någonstans i filträdet. Denna mapp kan du exempelvis döpa till "server". Efter du har skapat mappen vill vi installera alla serverfiler. Detta åstadkommer vi genom att skriva det följande i konsolen (om "server" var vad du kallade mappen): cd server, och därefter npm install json-server i samma kommandotolk. När vi har installerat serverns filer har vi dock inte fått med någon fil som faktiskt lagrar data. Denna behöver vi skapa på egen hand och sedan döpa till db.json. Notera filens tagg, .json, vilket är javascript format som primärt används för att skriva databaser. Dokumentet vi precis döpte till "db.json" behöver sedan placeras hierarkiskt längst upp i servermappen på samma ställe de två andra ".json" filerna, package.json och package-lock.json, vars uppgifter är att lagra serverns metadata. I vår nyskapade databasfil vill vi sedan formulera informationen som vi vill lagra. Eftersom vi tidigare skapade en bild och en rubrik, vilket hade kunnat tolkas som ett inlägg på en social media, hade vi kunnat skapa en databas som lagrar inläggens "likes". I så fall kan vi använda den följande json koden som skapar en array "likes" och ger array:en ett objekt som representerar en gillning.

    <VI. Uppdaterar komponenter i realtid med serverlagrad information/>

    Så hur bygger man då en React applikation som är beroende av en databas? Det första steget för att förbereda sin React applikation för att kommunicera med servrar är att installera ramverket "Axios". Detta ramverk möjliggör så att man med enkel syntax kan hämta och skriva data till json servrar. Axios är inte det enda ramverket som möjliggör detta, men det är en utav de mest lättanvända verktygen. Du installerar Axios genom att skriva cd react-app i kommandotolken för att välja din reactapplikations projektmapp, och sedan npm install axios för att ladda ner filerna. Nästa steg är att skapa en komponent som hämtar informationen från databasen vi skapade i förra genomgången och som sedan skriver ut antalet "likes" under den ursprungliga bilden. För att göra detta behöver du först skapa en ny javascript fil i "src" mappen i din React applikation. Denna fil kan du exempelvis döpa till "FetchAndPost". I filen skapar du sedan en komponent som returnerar en div med en text och en knapp inuti sig. Därefter exporterar du komponenten som default export default FetchAndPostLikes;, och importerar ramverken axios import axios from "axios"; samt useState och useEffect import {useState, useEffect} from "react";. Då kanske du tänker: "Vad skjutton är useState och useEffect?". Svaret är att de både har och göra med Hooks, som är en funktionell programmeringsteknik i React.

    Hooks gör det möjligt att använda statlig komponentlogik (till exempel att hålla reda på data eller hantera händelser) utan att behöva skapa en klasskomponent. Det ger oss möjlighet att skriva mer enhetlig och lättläst kod, eftersom vi slipper att skapa separata klasser för våra komponenter. Det finns olika sorters hooks, men de mest grundläggande är just useState och useEffect. useState låter oss hålla reda på data i våra komponenter, medan useEffect låter oss köra specifik kod när våra komponenter uppdateras eller monteras. På så sätt kan man hålla variabler uppdaterade under din applikations körtid, vilket är användbart för exempelvis sociala medier som hela tiden behöver hålla koll på inläggs gillningar m.m. Efter du har importerat dessa förnödenheter borde din fil och din komponent i alla fall se ut så här:

    Nästa moment är att skapa en hook som lagrar värdet på antalet likes under appens körtid. Det som gör hooks speciella är att de egentligen kan ses som en array som har två inbyggda variabler, en som man kallar på för att hämta dess lagrade värde, och en som man kallar på för att ändra hook:ens lagrade värde.

    När vi skapar vår hook skriver vi följaktligen const [likeNum, setLikeNum] = useState(0);, vilket skapar en state med standardvärdet 0 (dvs det värdet som den har innan den uppdateras). Hooken placerar vi sedan längst upp i vår nya komponent. Dess position i hierarkin är nämligen kritiskt för att allting ska fungera. Det finns exempelvis en hel del regler kring användandet av hooks. En av dessa regler är att man endast kan skapa och kalla på dem inuti React komponenter. En annan regel är att alla hooks man skapar måste vara placerade högst upp i komponenten.

                                 )
                                }

    Efter vi har skapat vår "likeNum" hook är nästa steg att hämta antalet likes från vår databas med Axios och spara värdet i likeNum genom att föra in det som en parameter i "setLikeNum". Det är även nu som useEffect blir relevant. Det är nämligen inuti den inbyggda useEffect som vi vill kalla på alla funktioner/kod som ska köras varje gång vår hook uppdateras.

    Vi börjar därför med att skriva useEffect(() => {//kalla på funktioner här}); under vår likeNum hook. Sedan lägger vi till "[likeNum]" innan slutparantesen, vilket meddelar React att all kod inuti useEffect ska köras varje gång värdet på likeNum förändras, dvs initialt då likeNum ges värdet 0 och varje gång därefter när användaren klickar på "gilla-knappen": useEffect(() => {//kalla på funktioner här}, [likeNum]);. Mellan useEffects klammerparanteser skapar vi sedan en asynkron (async) funktion kallad "getData" som vi sedan vill skriva all kod inuti. Varför vi skapar en async funktion och inte en normal javascript funktion (som automatiskt är synkronisk) beror på att vi inte vill stanna upp hela applikationen medan axios hämtar all data, en process som kan ta allting från 1ms till 10s beroende på hur mycket information som begärs. Genom att göra funktionen asynkron kan alla element som faktiskt är färdiga fortfarande skickas ut till DOM:en då "likeNum" förblir odefinierat, vilket gör att appen fortfarande upplevs som mer responsiv trots informationsbristen. Detta möjliggör också att man kan skriva ut "laddar..." bredvid texten som visar antalet likes då värdet på "likeNum" ännu inte har hämtats. Att skriva ut "laddar..." är dock inte jätteviktigt i vårt sammanhang eftersom vi hämtar väldigt lite information och följaktligen får en mycket snabb fetch request. Vi gör dock fortfarande funktionen asynkron eftersom det är så här man arbetar med hämtning av data i React vanligtvis. Innan vi går vidare bör din useEffect hook se ut så här:

    9. Tom useEffect hook

                                    useEffect(() => {
                              
                                 const getData = async () => {
                                  //skriv kod här
                                 };
                              
                                 getData();
                                }, [likeNum]);

    När vi hämtar vår data använder vi som tidigare förklarat ramverket axios. För att göra en "fetch request" (dvs hämta data) skriver man axios.get(//databasens address).then(response => {//hantera data här}). Eftersom databasen vi skapade i förra genomgången sparade alla gillningar i en array kallad "likes" blir vår databas-address http://localhost:3000/likes om servern körs på port 3000. Inuti klammerparanteserna av "axios.get" vill vi sedan uppdatera vår hook "likeNum". Detta åstadkommer vi genom att skriva setNumLikes(response.data.length). Vad denna kod gör är att uppdatera likeNum med värdet på antalet objekt i array:en "likes", vilket representerar antalet gillningar på inlägget. Texten response.data är således "likes" array:en i json form. Under .then() sektionen av kallelsen hittar vi även .catch(), vilket gör så att eventuella errormeddelanden skrivs ut till konsolen. Detta är mycket användbart för att identifiera eventuella fetch problem, så denna ska alltid skrivas ut när axios interagerar med din databas.

    10. Komplett useEffect hook

                                    useEffect(() => {
                              
                                 const getData = async () => {
                                          
                                  await axios.get("http://localhost:3000/likes")
                                  .then(response => {
                                   setLikeNum(response.data.length);
                                  }).catch(error => {
                                   console.log(error);
                                  });
                                 };
                              
                                 getData();
                                }, [likeNum]);

    Efter du har uppdaterat din useEffect hook är det dags för att förbereda "gilla-knappens" logik!

    Vad vi vill ska hända när man klickar på gillaknappen är att axios skickar ett objekt till vår databas och sparar den i "likes" array:en. Vi kan därför börja med att skapa en funktion som hanterar post-requesten. Själv döper jag funktionen till "handleLike". Nästa steg blir att skapa formatet för det objekt vi vill att axios ska leverera. Detta bestämmer vi genom att skapa ett objekt inuti funktionen som vi kan döpa till "like". Inuti detta objektet kan vi skriva vad som helst eftersom vi i denna genomgången inte kallar på "like" objektens innehåll, men för att få databasen att se hyfsat streamlined ut följer vi samma format som det urpsrungliga like objektet har. Dvs att ge objektet nyckeln "user" med värdet "Ny användare". Därefter skriver vi vår axios kallelse. Istället för att kalla på "axios.get()" kallar vi nu istället på "axios.post()". Den enda skillanden är att vi för in objektet vi skapade bredvid databasens address. Vi skickar således objektet genom axios.post(address, objekt), och uppdaterar återigen vår "likeNum" hook i .then() funktionen.

    11. Gillaknappens "post"-logik

                                    const handleLike = () => {

                                     const like = {
                                      user: "Ny användare"
                                     };
                                    
                                     axios.post("http://localhost:3000/likes", like)
                                      .then(response => {
                                       setLikeNum(response.data.length);
                                      }).catch(error => {
                                       console.log(error);
                                     });
                                }

    I slutändan måste vi också uppdatera HTML koden som returneras så att elementen faktiskt kan kommunicera med vår kod.

    Detta kan vi åstadkomma snabbt genom att ge bildrubriken texten "Likes:" och sedan infoga "likeNum" variabeln i textelementet genom två klammerparanteser. Angående knappelementet behöver vi endast ge den en "onClick" parameter med funktionen "handleLike" som värdet. Sedan är hela vår komponent färdigställd! Grattis!

    12. Anpassat returneringsblock

                                    return (
                                     <div>
                                      <p>Likes: {likeNum}</p>
                                      <button onClick={handleLike}>Like image</button>
                                     </div>
                                )

    Sista steget blir att importera vår komponent till "CardWithText" komponenten i CardView filen: import FetchAndPostLikes from "./FetchAndPost";, och sedan kalla på komponenten <FetchAndPostLikes /> nedanför bildens rubrik. Därefter är det bara att starta upp din React applikation och din server om du inte redan du har gjort det!
